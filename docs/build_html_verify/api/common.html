配置系统 (Configuration System)

HABIT 使用一套基于 YAML 的强类型配置系统。

Base configurator: shared infrastructure for domain-specific configurators.

V1 architecture: ServiceConfigurator has been split per business domain (Habitat / ML / Preprocessing). Each domain configurator extends BaseConfigurator and adds its own create_* factories.

This base class owns:

  • the configuration object,

  • the output directory derivation,

  • the logger creation policy (reuses the CLI-level logger if one is already set up via LoggerManager),

  • a small per-instance service cache.

Subclasses should override logger_name and add their own create_* factory methods. They MUST NOT bring in transitive imports from other business domains at module top level — defer those imports inside the factory methods to keep the import surface small.
