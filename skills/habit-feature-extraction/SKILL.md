# HABIT Feature Extraction

Extract per-subject quantitative features from previously generated habitat maps. Output is a set of CSV files ready for ML.

## Prerequisite

User must have already run `habit get-habitat` (see `habit-habitat-analysis` skill) and have:
- Habitat map files: `<habitats_map_folder>/<subject_id>/*_habitats_remapped.nrrd`
- Original images: `<raw_img_folder>/<subject_id>/images/` and `masks/`

If they don't have habitat maps yet, redirect to the `habit-habitat-analysis` skill first.

## CLI

```bash
habit extract --config <config_extract_features.yaml>
```

## Feature Types — Choose What to Extract

| Type | What | Output CSV | Requires PyRadiomics params |
|---|---|---|---|
| `traditional` | Classical radiomics on **original images** (whole-tumor ROI) | `raw_image_radiomics.csv` | Yes (`params_file_of_non_habitat`) |
| `non_radiomics` | Basic quantitative features (volumes, fractions) | `habitat_basic_features.csv` | No |
| `whole_habitat` | Radiomics on the entire **habitat label map** as one image | `whole_habitat_radiomics.csv` | Yes (`params_file_of_habitat`) |
| `each_habitat` | Radiomics for **each habitat** independently — generates many features | `each_habitat_radiomics.csv` | Yes |
| `msi` | **Most Significant Intensity** — habitat-specific intensity stats | `msi_features.csv` | No |
| `ith_score` | **Intra-Tumor Heterogeneity** scores (entropy-based) | `ith_scores.csv` | No |

### Recommendation by Use Case

- **Standard publication setup**: `[traditional, non_radiomics, whole_habitat, msi, ith_score]`
- **Maximum features (slow, high-dim)**: add `each_habitat`
- **Quick exploration**: `[non_radiomics, msi, ith_score]` (no PyRadiomics needed)

## Required Files

When using radiomics-based features, user needs PyRadiomics parameter files:

```yaml
params_file_of_non_habitat: ./config/parameter.yaml         # for traditional
params_file_of_habitat: ./config/parameter_habitat.yaml     # for whole/each habitat
```

A `parameter.yaml` example (PyRadiomics standard):
```yaml
imageType:
  Original: {}

featureClass:
  firstorder:
  shape:
  glcm:
  glrlm:
  glszm:

setting:
  binWidth: 25
  resampledPixelSpacing: [1, 1, 1]
```

If user doesn't have these, point them to the official PyRadiomics docs:
https://pyradiomics.readthedocs.io/en/latest/customization.html

## Standard Config

```yaml
params_file_of_non_habitat: ./config/parameter.yaml
params_file_of_habitat: ./config/parameter_habitat.yaml

raw_img_folder: ./data/preprocessed_images
habitats_map_folder: ./results/habitat_analysis
out_dir: ./results/features

n_processes: 4
habitat_pattern: '*_habitats_remapped.nrrd'

feature_types:
  - traditional
  - non_radiomics
  - whole_habitat
  - msi
  - ith_score

n_habitats:           # leave empty = auto-detect
debug: false
```

## Voxel-based GLCM Auto-Protection

When using voxel-level radiomics with small kernels (kernelRadius=1-3), GLCM features can fail on overly homogeneous neighborhoods. HABIT automatically restricts GLCM to 4 robust features:
- `Contrast`, `Correlation`, `JointEnergy`, `Idm`

This is silent and intentional. Tell users about it if they see GLCM warnings.

If users want full GLCM, they should:
- Increase `kernelRadius` (e.g. 2 or 3)
- OR explicitly list GLCM features in their `parameter.yaml`

## Common Pitfalls

1. **`habitat_pattern` mismatch** → check actual file names in `habitats_map_folder/<subject>/`. Default is `*_habitats_remapped.nrrd`. Use `*_habitats.nrrd` for one_step output.
2. **Missing PyRadiomics params** → fail fast; tell user which features need a params file.
3. **`each_habitat` produces 5×N features for 5 habitats** → can balloon to 1000+ features. Only use if user has large cohort (>200 patients) or aggressive feature selection downstream.
4. **Subject ID mismatch** between `raw_img_folder` and `habitats_map_folder` → all subjects must have BOTH. Check folder naming consistency.
5. **MSI/ITH require habitat maps** but no PyRadiomics — quickest path for users without PyRadiomics.

## Output Structure

```
out_dir/
├── raw_image_radiomics.csv          # if 'traditional'
├── habitat_basic_features.csv       # if 'non_radiomics'
├── whole_habitat_radiomics.csv      # if 'whole_habitat'
├── each_habitat_radiomics.csv       # if 'each_habitat'
├── msi_features.csv                 # if 'msi'
├── ith_scores.csv                   # if 'ith_score'
└── extract_features.log
```

Each CSV has subject IDs in the first column, features in subsequent columns. **Direct input** for `habit model` (machine learning).
