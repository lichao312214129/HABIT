# HABIT 便携包（conda pack）打包说明

面向维护者。脚本位于本目录 `developer/`。

| 平台 | 一键打包 | Staging | 用户 setup |
|------|----------|---------|------------|
| Windows | `pack_habit.bat` | `stage_external_tools.bat` | `setup_habit.bat` |
| macOS / Linux | `pack_habit.sh` | `stage_external_tools.sh` | `setup_habit.sh` |

**重要**：必须在 **与目标用户相同的操作系统** 上打包（Windows 包不能在 Linux/macOS 用，反之亦然）。

---

## 一、通用准备（所有平台）

```bash
conda create -n habit python=3.10
conda activate habit
conda install -c conda-forge conda-pack

pip install -r requirements.txt
pip install .                    # 必须用 pip install . ，不要用 pip install -e .

habit --version
```

平台差异见下文各节。

---

## 二、Windows 打包

### 环境要求

- Windows 64 位
- `demo_data/` 下须存在：`dcm2niix.exe`、`elastix.exe`、`transformix.exe`

### 依赖安装（Windows 示例）

```bash
conda activate habit
cd /d <HABIT 仓库根目录>

pip install pyradiomics-3.0.1-cp310-cp310-win_amd64.whl

# GPU 版（任选其一）
pip install -r requirements-gpu.txt

pip install .
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
```

### 一键打包

```bash
developer\pack_habit.bat
developer\pack_habit.bat HABIT-win-py310-gpu-v0.1.0.tar.gz
```

默认输出：`HABIT-win-py310-gpu-v0.1.0.tar.gz`

### Staging 复制内容

| 目标 | 文件 |
|------|------|
| 环境根目录（与 `python.exe` 同级） | `setup_habit.bat`、`install_gpu_torch.bat` |
| `Scripts/` | `dcm2niix.exe`、`elastix.exe`、`transformix.exe` |

### 用户安装

1. 新建空文件夹（如 `D:\habit-cpu`），将 `.tar.gz` / `.zip` 放入后 **解压到当前目录**（`python.exe` 与 `setup_habit.bat` 同级）
2. 运行 `setup_habit.bat` → 新开终端 → `habit --version`
3. CPU 包 + NVIDIA：网盘 wheel 放入 pack 根目录 → `install_gpu_torch.bat`

## 三、Linux 打包

### 环境要求

- Linux x86_64（与目标用户机器一致）
- 外部工具（二选一）：
  1. **推荐**：打包前装入 conda 环境  
     `conda install -c conda-forge dcm2niix`  
     （elastix 可用 conda-forge / 官方二进制，装好后已在 `bin/` 内，会随 pack 带走）
  2. 或将无扩展名可执行文件放入 `demo_data/linux/`：`dcm2niix`、`elastix`、`transformix`（staging 会复制到 `bin/`）

### 依赖安装（Linux 示例）

```bash
conda activate habit
cd <HABIT 仓库根目录>

pip install pyradiomics

# GPU（NVIDIA + CUDA 12.1）
pip install -r requirements-gpu.txt

pip install .

# 可选：随 pack 捆绑 CLI 工具
conda install -c conda-forge dcm2niix
```

### 一键打包

```bash
chmod +x developer/*.sh
bash developer/pack_habit.sh
bash developer/pack_habit.sh HABIT-linux-py310-gpu-v0.1.0.tar.gz
```

默认输出：`HABIT-linux-py310-gpu-v0.1.0.tar.gz`

### 解压与测试（维护者自测）

```bash
mkdir -p ~/HABIT-test && tar -xf HABIT-linux-py310-gpu-v0.1.0.tar.gz -C ~/HABIT-test
cd ~/HABIT-test
bash setup_habit.sh
source .habit_path.sh
habit --version
```

### Staging 复制内容

| 目标 | 文件 |
|------|------|
| 环境根目录 | `setup_habit.sh`、`install_gpu_torch.sh`、`requirements-gpu-torch-only.txt` |
| `bin/` | `demo_data/linux/` 下可选工具；或 conda 预装工具 |

### 用户安装

```bash
tar -xf HABIT-linux-py310-gpu-v0.1.0.tar.gz -C ~/HABIT
cd ~/HABIT
bash setup_habit.sh
source ~/.bashrc   # 或 source .habit_path.sh
habit --version
```

## 四、macOS 打包

### 环境要求

- macOS（Apple Silicon 与 Intel 需 **分别** 打包）
- 外部工具：同 Linux，可用 `conda install -c conda-forge dcm2niix`，或放入 `demo_data/darwin/`
- **无 NVIDIA GPU**：便携包一般为 **CPU 版** torch；`install_gpu_torch.sh` 在 macOS 上通常不需要

### 依赖安装（macOS 示例）

```bash
conda activate habit
cd <HABIT 仓库根目录>

pip install pyradiomics
pip install .

conda install -c conda-forge dcm2niix
```

### 一键打包

```bash
chmod +x developer/*.sh
bash developer/pack_habit.sh
bash developer/pack_habit.sh HABIT-macos-py310-cpu-v0.1.0.tar.gz
```

默认输出：`HABIT-macos-py310-cpu-v0.1.0.tar.gz`

### 用户安装

```bash
tar -xf HABIT-macos-py310-cpu-v0.1.0.tar.gz -C ~/HABIT
cd ~/HABIT
bash setup_habit.sh
source ~/.zshrc       # 或 source .habit_path.sh
habit --version
```

## 五、分步打包（可选，Unix）

```bash
conda activate habit
bash developer/stage_external_tools.sh
conda pack -n habit -o HABIT-linux-py310-gpu-v0.1.0.tar.gz --compress-level 9
```

Windows 分步：

```bash
developer\stage_external_tools.bat
conda pack -n habit -o HABIT-win-py310-gpu-v0.1.0.tar.gz --compress-level 9
```

## 六、发布给用户

1. 按平台分别上传 tar.gz（命名建议含 `win` / `linux` / `macos`、`py310`、cpu/gpu、版本号）
2. 告知用户运行对应 `setup_habit.bat` 或 `setup_habit.sh`
3. `config/` 不在包内，用户从 GitHub ZIP 复制
4. 用户文档：`docs/source/getting_started/installation_zh.rst`「方式一」（当前以 Windows 便携包为主）

## 七、常见问题

**Q: `pip install -e .` 可以吗？**

A: 不可以。editable 会链到源码路径，用户解压后 `import habit` 失败。

**Q: Windows PowerShell 报「conda 不是内部或外部命令」？**

A: 使用新版 `pack_habit.bat`（自动查找 `conda.exe`），或改用 Anaconda Prompt。

**Q: Linux/macOS 没有 `demo_data/*.exe`？**

A: Windows 的 exe 不能用于 Unix。请用 conda 装入工具，或把 Unix 二进制放到 `demo_data/linux/` / `demo_data/darwin/`。

**Q: 打包前改了 torch，用户端 conda-unpack 报错？**

A: 固定 torch 版本后再 pack；不要在 pack 前反复升级 torch。

**Q: 每次发新版都要重新 pack 吗？**

A: 是。`pip install .` → 对应平台的 `pack_habit.bat` / `pack_habit.sh` → 上传新 tar.gz。

**Q: 能否在 Windows 上打 Linux 包？**

A: 不能。conda pack 与平台绑定，必须在目标 OS 上打包。
